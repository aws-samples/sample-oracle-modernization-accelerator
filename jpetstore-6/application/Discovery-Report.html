🔧 Query Conversion Examples

1. Oracle 스타일 조인 (콤마 조인) 변환

Description: Oracle의 콤마 조인 구문을 ANSI JOIN으로 변환

SELECT ACCOUNT.EMAIL, PROFILE.LANGPREF, BANNERDATA.BANNERNAME FROM ACCOUNT, PROFILE, SIGNON, BANNERDATA WHERE ACCOUNT.USERID = #{username} AND SIGNON.USERNAME = ACCOUNT.USERID AND PROFILE.USERID = ACCOUNT.USERID AND PROFILE.FAVCATEGORY = BANNERDATA.FAVCATEGORY
SELECT ACCOUNT.EMAIL, PROFILE.LANGPREF, BANNERDATA.BANNERNAME FROM ACCOUNT INNER JOIN SIGNON ON SIGNON.USERNAME = ACCOUNT.USERID INNER JOIN PROFILE ON PROFILE.USERID = ACCOUNT.USERID INNER JOIN BANNERDATA ON PROFILE.FAVCATEGORY = BANNERDATA.FAVCATEGORY WHERE ACCOUNT.USERID = #{username}
💡 Conversion Notes: 콤마 조인을 명시적인 INNER JOIN으로 변환하여 가독성과 성능을 향상시킵니다.

2. 다중 테이블 조인 변환

Description: 3개 테이블의 콤마 조인을 ANSI JOIN으로 변환

SELECT ITEM.ITEMID, ITEM.LISTPRICE, INVENTORY.QTY AS quantity FROM ITEM, INVENTORY, PRODUCT WHERE PRODUCT.PRODUCTID = ITEM.PRODUCTID AND ITEM.ITEMID = INVENTORY.ITEMID AND ITEM.ITEMID = #{value}
SELECT ITEM.ITEMID, ITEM.LISTPRICE, INVENTORY.QTY AS quantity FROM ITEM INNER JOIN PRODUCT ON PRODUCT.PRODUCTID = ITEM.PRODUCTID INNER JOIN INVENTORY ON ITEM.ITEMID = INVENTORY.ITEMID WHERE ITEM.ITEMID = #{value}
💡 Conversion Notes: 조인 조건을 명확히 하여 쿼리 실행 계획 최적화에 도움이 됩니다.

⚠️ Migration Considerations

  • 조인 성능: ANSI JOIN으로 변환 시 실행 계획 검토 필요
  • 데이터 타입: Oracle과 PostgreSQL 간 데이터 타입 호환성 확인
  • 함수 호환성: 대부분의 표준 SQL 함수는 호환 가능
