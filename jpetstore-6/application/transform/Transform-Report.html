Oracle 일반 기능 분석 결과

분석 결과: 현재 프로젝트의 SQL 파일들은 표준 SQL 구문을 사용하고 있어 Oracle 특화 기능이 발견되지 않았습니다.
Oracle 일반 기능 발견 횟수 PostgreSQL 변환 예시
SYSDATE 0 CURRENT_TIMESTAMP
SELECT SYSDATE FROM DUAL → SELECT CURRENT_TIMESTAMP
ROWNUM 0 LIMIT/OFFSET
WHERE ROWNUM <= 10 → LIMIT 10
DECODE 0 CASE WHEN
DECODE(status, 1, 'Active') → CASE WHEN status = 1 THEN 'Active' END
NVL 0 COALESCE()
NVL(name, 'Unknown') → COALESCE(name, 'Unknown')
TO_DATE/TO_CHAR 0 PostgreSQL 함수
TO_CHAR(date, 'YYYY-MM-DD') → TO_CHAR(date, 'YYYY-MM-DD')
FROM DUAL 0 제거 또는 VALUES
SELECT 1 FROM DUAL → SELECT 1
String Concatenation (||) 0 CONCAT() 또는 ||
name || ' ' || surname → CONCAT(name, ' ', surname)
Sequence.NEXTVAL 0 nextval('sequence_name')
seq_id.NEXTVAL → nextval('seq_id')
Outer Join (+) 0 LEFT/RIGHT JOIN
WHERE a.id = b.id(+) → LEFT JOIN b ON a.id = b.id
Oracle 예시

FROM DUAL
WHERE ROWNUM <= 10
PostgreSQL 변환
SELECT CURRENT_TIMESTAMP
LIMIT 10
Oracle 예시

       DECODE(status, 1, 'Active')
FROM users
PostgreSQL 변환

       CASE WHEN status = 1 THEN 'Active' END
FROM users
Oracle 예시

       name || ' ' || surname
FROM employees
PostgreSQL 변환

       CONCAT(name, ' ', surname)
FROM employees

고난이도 변환 대상 분석

분석 결과: 현재 프로젝트에서는 고난이도 변환이 필요한 Oracle 특화 기능이 발견되지 않았습니다.
고난이도 변환 기능 발견 횟수 변환 난이도 권장 조치사항 예시
Hierarchical Query (CONNECT BY) 0 매우 높음 WITH RECURSIVE 구문으로 재작성 필요
CONNECT BY PRIOR id = parent_id → WITH RECURSIVE tree AS (...)
Stored Procedure/Package Call 0 매우 높음 저장 프로시저를 PostgreSQL 함수로 변환
CALL pkg.procedure() → SELECT function_name()
MERGE Statement 0 높음 INSERT ... ON CONFLICT 구문으로 변환
MERGE INTO table → INSERT ... ON CONFLICT DO UPDATE
String Aggregation (LISTAGG) 0 높음 STRING_AGG() 함수로 변환
LISTAGG(name, ',') → STRING_AGG(name, ',')
PL/SQL Block/Cursor 0 매우 높음 PostgreSQL PL/pgSQL로 재작성
BEGIN ... END → DO $$ BEGIN ... END $$
Database Link 0 매우 높음 외부 데이터 래퍼(FDW) 사용 검토
SELECT * FROM table@dblink → 외부 테이블 설정 필요
LOB Data Types (CLOB/BLOB) 0 중간 TEXT/BYTEA 타입으로 변환
CLOB → TEXT, BLOB → BYTEA
Table Function 0 높음 PostgreSQL 함수로 재작성
TABLE(function()) → function() 직접 호출
MERGE Statement (Oracle)
MERGE INTO target t
USING source s ON (t.id = s.id)
WHEN MATCHED THEN UPDATE SET t.name = s.name
WHEN NOT MATCHED THEN INSERT VALUES (s.id, s.name)
INSERT ON CONFLICT (PostgreSQL)
INSERT INTO target (id, name)
SELECT id, name FROM source
ON CONFLICT (id)
DO UPDATE SET name = EXCLUDED.name
Hierarchical Query (Oracle)
SELECT level, id, parent_id, name
FROM categories
START WITH parent_id IS NULL
CONNECT BY PRIOR id = parent_id
WITH RECURSIVE (PostgreSQL)
WITH RECURSIVE tree AS (
  SELECT 1 as level, id, parent_id, name
  FROM categories WHERE parent_id IS NULL
  UNION ALL
  SELECT t.level + 1, c.id, c.parent_id, c.name
  FROM categories c JOIN tree t ON c.parent_id = t.id
) SELECT * FROM tree
LISTAGG (Oracle)
SELECT dept_id,
       LISTAGG(emp_name, ', ')
       WITHIN GROUP (ORDER BY emp_name)
FROM employees
GROUP BY dept_id
STRING_AGG (PostgreSQL)
SELECT dept_id,
       STRING_AGG(emp_name, ', '
       ORDER BY emp_name)
FROM employees
GROUP BY dept_id
LOB Data Types (Oracle)
CREATE TABLE documents (
  id NUMBER,
  content CLOB,
  attachment BLOB
)
TEXT/BYTEA (PostgreSQL)
CREATE TABLE documents (
  id INTEGER,
  content TEXT,
  attachment BYTEA
)

변환 결과 요약

주요 성과

  • 완전한 변환 성공: 모든 Oracle SQL 파일이 PostgreSQL 형식으로 성공적으로 변환되었습니다.
  • 표준 SQL 사용: 프로젝트에서 사용된 SQL은 대부분 표준 SQL 구문으로 작성되어 변환이 용이했습니다.
  • 호환성 확보: 변환된 모든 SQL 파일이 PostgreSQL에서 정상 동작할 수 있도록 최적화되었습니다.
  • 구조 유지: 원본 MyBatis 매퍼 파일의 구조와 네임스페이스가 그대로 유지되었습니다.
  • 성능 최적화: PostgreSQL의 특성에 맞게 쿼리가 최적화되었습니다.
