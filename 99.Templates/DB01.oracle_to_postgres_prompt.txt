다음 Oracle {object_type} DDL을 PostgreSQL 호환 구문으로 변환:

{oracle_ddl}

요구사항:
1. PostgreSQL 변환 코드만 제공 (설명, 로그, 마크다운 포맷 제외)
2. 모든 문장은 세미콜론(;)으로 종료
3. 가독성을 위한 들여쓰기 적용
4. 다음 Oracle 기능의 PostgreSQL 매핑 준수:
   - PIVOT → CASE 및 집계 함수 사용
   - LISTAGG → STRING_AGG
   - EXECUTE IMMEDIATE → EXECUTE
   - 패키지 → 스키마 및 함수 조합
   - 시퀀스 → PostgreSQL 시퀀스 구문
   - 데이터 타입 변환:
     * VARCHAR2 → VARCHAR
     * NUMBER → NUMERIC
     * CLOB → TEXT
     * DATE → TIMESTAMP
     * RAW → BYTEA
   - Oracle 힌트 제거 또는 대체
5. 표준 ANSI SQL 준수
6. PostgreSQL 13 이상 버전 호환성 보장

변환 시 주의사항:
- PL/SQL 블록을 PL/pgSQL로 올바르게 변환
- 오라클 고유 함수를 PostgreSQL 대응 함수로 변환
- 트랜잭션 관리 구문 적절히 조정
- 인덱스 및 제약조건 구문 호환성 확인
- 파티셔닝 구문 PostgreSQL 방식으로 변환
